# PRD — 썸네일 테스터 앱 (B2C, 캐시워크형)

> **Product Name (가칭):** ThumbLab Tester
> **Target:** 유튜브를 즐겨 보는 일반 사용자 (앱테크/리워드 사용자 포함)
> **Platform:** iOS / Android (React Native + Expo, 또는 Next.js + Capacitor)
> **Backend 공유:** B2B 웹과 동일 Next.js (App Router) API
> **검증:** Zod
> **버전:** v1.0 (MVP)
> **작성일:** 2026-05-16

---

## 1. 제품 개요

### 1.1 한 줄 정의
유튜브 썸네일 두 개를 보고 더 클릭하고 싶은 쪽을 고르면 **코인을 적립**하는 리워드 앱.

### 1.2 사업 모델 상의 역할
- B2B(에이전시)가 의뢰한 썸네일 테스트의 **응답자(시청자 풀)** 역할
- 패널의 인구통계·시청 카테고리 정합성과 **응답 신뢰도**가 곧 제품 품질
- 캐시워크/타임스프레드와 유사한 일상 적립 UX로 **체류 시간 짧고 자주 열게 함**

### 1.3 핵심 가치 제안
| 사용자에게 | 우리에게 |
|---|---|
| 5초만에 끝나는 가벼운 적립 미션 | 카테고리 매칭된 양질의 패널 확보 |
| 유튜브 보던 흐름을 흉내낸 자연스러운 UX | 정직한 응답 시그널(시간·위치·일관성) |
| 적중 보너스로 게이미피케이션 | 다단계 신뢰도 필터로 신호/잡음 분리 |

## 3. 기능 요구사항

### 3.1 회원가입 / 정보 수집

**가입 흐름**
1. 휴대폰 번호 인증 또는 카카오·애플·구글 소셜 로그인
2. 기본 프로필
   - 닉네임
   - 성별 (남/여/응답 안 함)
   - 생년 (연 단위만 — 연령대 산출용)
3. 시청 성향 설문 (필수)
   - 자주 보는 카테고리 **3~5개 선택** (게임/뷰티/푸드/브이로그/교육/IT·테크/키즈/음악/스포츠/엔터·예능 등)
   - 하루 유튜브 시청 시간 (30분 미만/1시간/2시간/3시간 이상)
   - 주로 보는 디바이스 (모바일/PC/TV)
4. 알림 권한 + 마케팅 동의
5. **온보딩 튜토리얼**: 샘플 테스트 1회 진행 → 즉시 50코인 지급

**Zod 스키마**
```ts
export const TesterProfileSchema = z.object({
  nickname: z.string().min(2).max(20),
  gender: z.enum(["M","F","UNDISCLOSED"]),
  birthYear: z.number().int().min(1940).max(new Date().getFullYear()-13),
  categories: z.array(z.string()).min(3).max(5),
  dailyWatchTime: z.enum(["<30m","~1h","~2h","3h+"]),
  device: z.enum(["MOBILE","PC","TV","MIXED"]),
});
```

**중요**
- 14세 미만 가입 차단 (개인정보보호법)
- 한 디바이스/한 전화번호 = 1계정 (디바이스 핑거프린트 + SMS 인증)
- 카테고리는 분기마다 재확인 알림 (시청 취향 변화 반영)

### 3.3 썸네일 테스트 진행 (핵심)

**환경 설계 — 유튜브 모방**
- 화면 상단: "어떤 영상을 클릭하시겠어요?" 한 줄 안내
- 본문: 유튜브 홈 그리드를 모방한 카드 2~4개
  - 썸네일 이미지 (16:9)
  - 가짜 제목 (실제 영상 제목 노출 금지 — 썸네일만으로 판단하게)
    - 또는 동일 더미 제목으로 통일 ("새 영상" 같은 중립 텍스트)
  - 가짜 채널명·구독자수·업로드 시각 (모두 동일하게 통제)
  - 가짜 조회수도 동일 또는 미표시
- 탭하면 그 시안에 한 표
- 탭 후 **즉시 다음 테스트로 자동 진행** (피드 형식, 무한 스크롤 X · 카드 슬라이드 O)

**측정하는 행동 시그널 (서버로 전송)**
```ts
export const VoteSubmitSchema = z.object({
  testId: z.string().cuid(),
  testerId: z.string().cuid(),
  chosenThumbnailId: z.string().cuid(),
  responseTimeMs: z.number().int().positive(),       // 표시 → 탭까지
  tapPosition: z.object({ x: z.number(), y: z.number() }),
  thumbnailOrder: z.array(z.string()),                // 노출 순서 (랜덤화됨)
  appSessionId: z.string().uuid(),
  deviceFingerprint: z.string(),
  clientTimestamp: z.string().datetime(),
});
```

**노출 통제**
- 시안 순서는 매 응답마다 **랜덤 셔플** (1번 위치 편향 방지)
- 한 사용자는 같은 테스트를 한 번만 진행
- 카테고리 매칭이 안 된 테스트는 피드에 노출되지 않음

**테스트가 떨어졌을 때**
- "오늘의 미션 끝! 다음 테스트는 ⏱️ 약 N시간 후에 도착해요"
- 푸시 알림 옵션

### 3.4 응답 신뢰도 평가 (4단 필터)

이 모듈은 **서버 측**에서 동작. 앱은 시그널만 충실히 보낸다.

| 필터 | 메커니즘 | 액션 |
|---|---|---|
| **F1. 행동 패턴** | - 최근 20표 중 동일 위치 클릭률 70%↑<br>- 응답시간 표준편차가 비정상적으로 낮음(로봇 의심) | 신뢰도 점수 -10, 누적 시 가중치 하향 |
| **F2. 어텐션 체크** | 한쪽이 명백히 우월한 컨트롤 쌍을 **10~15회마다 1회** 끼워넣음. (예: 정상 썸네일 vs 검은 화면)<br>틀리면 페널티 | 1회 실패: 경고<br>2회 연속: 해당 세션 데이터 제외<br>3회 연속: 24h 응답 차단 |
| **F3. 응답 시간** | 정상 구간 3~15초. 1초 미만 또는 30초 초과 의심표 마킹 | 의심표는 집계 가중치 0.3, 누적 시 신뢰도 하향 |
| **F4. 결과 정합성** | 완료된 테스트의 다수파 적중률 누적 (베이지안 업데이트) | 신뢰도 점수에 반영, 적중률 낮은 사용자는 응답 가중치 자동 하향 (최저 0.2) |

**신뢰도 등급화 (사용자에게도 표시)**
- S (95~100): 가중치 1.2, 보너스 상시
- A (80~94): 가중치 1.0
- B (60~79): 가중치 0.7
- C (60 미만): 가중치 0.3, 출금 한도 축소

**투명성 원칙**
- 사용자에게 "현재 등급", "왜 차감/적립됐는지" 사유는 보여주되 **정확한 알고리즘 임계값은 비공개**(어뷰징 방지)
- 이의제기 채널 제공

### 3.5 코인 적립 / 차감 / 출금

**적립**
| 액션 | 지급 코인 |
|---|---|
| 일반 응답 1회 | **5 코인** (기본) × 신뢰도 가중치 |
| 어텐션 체크 정답 | +3 코인 |
| 완료된 테스트의 다수파 적중 | +10 코인 (테스트 종료 후 일괄 정산) |
| 출석 보상 (일 1회 최초 응답) | +20 코인 |
| 친구 초대 (초대 받은 친구 첫 응답 완료) | +500 코인 |

> 1코인 = 1원 가치 기준. 비용 계산은 별도 `Pricing_Strategy.md`에서.

**차감**
| 조건 | 차감 |
|---|---|
| 어텐션 체크 실패 (단일) | -5 코인 |
| 24시간 내 어텐션 2회 연속 실패 | -20 코인 + 해당일 보너스 무효 |
| 부정 응답 패턴 판정 (F1 임계 초과) | -50 코인 + 24h 차단 |

**출금**
- 5,000 코인부터 출금 신청 가능
- 출금 옵션
  - 현금 (계좌이체) — 수수료 100원
  - 기프티콘 (편의점/카페) — 5,000 / 10,000 / 30,000원권
  - 카카오페이/네이버페이 포인트 전환
- 신뢰도 C 등급은 일 1만 코인까지만 출금 가능
- 출금 신청 → KYC 본인 확인 (1회) → 영업일 기준 3일 내 지급

## 4. 어뷰징 방지 (앱 차원)

- 디바이스 핑거프린팅 (IDFA/AAID + WebGL/Canvas signature)
- 동일 IP/디바이스 다중 계정 차단
- VPN/에뮬레이터 탐지 (조건부 차단 — 신뢰도 점수에만 반영)
- 루팅/탈옥 단말 응답 가중치 하향
- SMS/카카오 인증 + 본인 명의 휴대폰만 허용
- 출금 시 KYC

## 6. 기술 / 시스템

```
[Mobile App: React Native + Expo]
        │ HTTPS, JWT
        ▼
[Next.js API (App Router)] ── shared with B2B web
   • Zod 검증
   • 신뢰도 점수 계산
   • 매칭 알고리즘 (테스트 ↔ 테스터)
        │
        ▼
[Postgres + Redis(랭킹/캐시)]
        │
        ▼
[Workers]
   • 테스트 종료 / 정산 / 정합성 평가
   • 푸시 발송 (FCM/APNs)
```

**매칭 알고리즘 요지**
- 테스트의 audience 조건 ∩ 테스터 프로필
- 매칭된 테스터 풀 중 신뢰도 가중치 합이 목표 투표 수에 도달할 때까지 피드 노출
- 같은 테스터에게는 같은 테스트 1회만

## 8. 보안 / 프라이버시
- 개인정보(생년, 카테고리)는 집계 형태로만 에이전시에 노출 (개별 식별 불가)
- 썸네일 이미지는 앱 측 로컬 캐시 후 일정 시간 뒤 삭제
- 결과 페이지에서 어떤 시안이 1등인지는 응답한 테스터에게도 공개하지 않음 (담합 방지)
- 만 14세 미만 가입 차단, 만 19세 미만은 출금 시 보호자 동의
